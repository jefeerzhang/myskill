# Material-Based Research Ideation

## 输出路径规范
**默认输出目录**：`./研究构想/<材料名称>/`
**文件编码**：UTF-8 无 BOM

当用户指定一个文件夹或要求基于某些材料提出研究假设时，请严格执行以下工作流：

## 1. 材料扫描与深度解析 (Information Extraction)

### 1.1 遍历文件夹
首先询问用户材料文件夹的路径。然后使用文件系统工具遍历指定文件夹：

```bash
ls -la /path/to/folder
```

识别所有文档文件并分类处理：
- **PDF** (`.pdf`) → **必须使用 `pdf` 技能**（调用 pdfplumber 或 pypdf）提取文本
- **Word** (`.docx`, `.doc`) → **必须使用 `docx` 技能** 提取内容
- **Markdown/文本** (`.md`, `.txt`) → 使用 `read` 工具直接读取

**重要提示**：如果文件夹内有超过 5 个 PDF 文件，建议先批量提取到一个临时目录，再统一阅读。可以使用如下 Python 脚本：

```python
import pdfplumber
from pathlib import Path

pdf_dir = Path("/path/to/folder")
for pdf in pdf_dir.glob("*.pdf"):
    with pdfplumber.open(pdf) as p:
        text = "\n".join([page.extract_text() for page in p.pages])
        (pdf_dir / f"{pdf.stem}.txt").write_text(text, encoding="utf-8")
```

### 1.2 提炼核心事实
对每份材料，提取以下关键信息并整理成表格：

| 维度 | 提取内容 |
|------|----------|
| **外生冲击/政策事件** | 有无特定的政策出台、制度变革、自然实验？（时间、名称、覆盖范围） |
| **微观主体** | 研究对象是谁？（企业、农户、地方政府、家庭、个人等） |
| **核心变量** | 解释变量 (X)、被解释变量 (Y)、潜在机制变量 (M) |
| **研究情境** | 时间、地点、行业、制度背景 |
| **研究方法** | 材料原文使用的识别策略（若有） |
| **主要发现** | 材料报告的核心结论（若有） |

**输出要求**：用 300-500 字总结所有材料的共同主题和差异点。

## 1.3 RAG 知识库检索（提升研究问题准确性）

### 为什么要做 RAG 检索？
材料提取的是单篇文献的核心事实，但学术研究需要站在领域前沿。本步骤通过检索用户本地知识库，获取：
- 相关理论基础的经典文献
- 研究领域的最新进展
- 已有研究的空白和争议点
- 因果识别的常用方法

## 2. 生成 3-5 个实证研究问题 (Research Questions)

完全基于你读取到的材料，提出递进式的研究问题：

### RQ 类型框架
1. **特征事实 (Stylized Facts)**：材料中展现了怎样的结构性规律或相关性模式？
2. **因果效应 (Causal Effect)**：材料中提到的某项举措/特征，对核心结果变量有何因果影响？
3. **作用机制 (Mechanism)**：这种影响是通过什么渠道传导的？
4. **异质性分析 (Heterogeneity)**：效应在不同群体/情境下有何差异？
5. **福利/政策含义 (Welfare/Policy Implications)**：研究结论对政策设计有何启示？

## 3. 为每个问题搭建实证分析框架 (Empirical Strategy)

针对上述每一个 RQ，你必须以学术审稿人的严谨度输出以下结构：

### 3.1 核心假设 (Hypothesis)
提出一个明确、可证伪的预测，指出预期的符号方向（正/负/倒 U 型等）。
- 格式示例：`H1: XXX 显著促进了 YYY` 或 `H2: 在 ZZZ 情境下，XXX 对 YYY 的影响更强`

### 3.2 识别策略 (Identification Strategy)
**推荐最合适的计量方法**，从以下中选择并说明理由：
- **双重差分 (DiD)**：适用于政策冲击有明确处理组/对照组、多期面板
- **多期 DiD / 交叠 DiD**：处理时间在不同个体间交错
- **断点回归 (RDD)**：存在连续分配变量和明确阈值
- **工具变量 (IV)**：存在与内生变量相关但与被解释变量无关的外生工具
- **回归控制函数 / Heckman 选择模型**：样本选择偏误
- **多维固定效应面板模型**：控制个体、时间、行业等多维异质性
- **合成控制法 (Synthetic Control)**：单一处理单元的政策评估

**明确指出**：
- Treatment 变量是什么（如何定义处理组）
- Control 组是谁（如何构建反事实）
- 识别假设是什么（如平行趋势假设）

### 3.3 数据需求 (Data Requirements)
验证该假设需要收集的具体数据字段，**必须包含以下内容**：

**数据来源建议（中国情境）**：
| 数据类型 | 推荐数据库 | 具体表名/字段 |
|----------|------------|---------------|
| 上市公司财务 | CSMAR | 财务指标分析库、资产负债表、利润表、现金流量表 |
| 上市公司治理 | CSMAR | 公司治理库、股权结构库、董监高个人信息库 |
| 专利/创新 | CNIPA、CNRDS | 专利申请数据库、绿色专利 IPC 分类 |
| 微观调查 | CHFS、CFPS、CEPS | 家庭金融调查、中国家庭追踪调查、中国教育追踪调查 |
| 城市/省级宏观 | 中国城市统计年鉴、EPS | 地级市社会经济数据 |
| 海关/贸易 | 中国海关数据库 | 企业 - 产品 - 国别层面进出口数据 |
| 工业企业 | 中国工业企业数据库 | 全部国有及规模以上工业企业数据 |

**输出格式要求**：
- **核心变量**：Y、X 的度量方式（公式）及数据来源（具体到数据库表名）
- **控制变量**：需要控制的混淆因素（至少列出 5 个常见控制变量）
- **机制检验**：中介变量或调节变量的度量方式
- **数据类型**：微观调查数据/宏观加总数据/企业级数据/卫星数据等
- **样本期建议**：基于数据可得性给出建议（如"2007 年新会计准则后数据更可靠"）

### 3.4 数据可得性检查清单（Data Availability Checklist）

针对上述每个核心变量，必须填写以下检查清单：

| 变量 | 主数据库 | 备选数据库 | 获取成本 | 样本覆盖 | 收集难度 |
|------|----------|------------|----------|----------|----------|
| Y: [变量名] | [数据库_表名] | [备选] | 免费/已购/需购买 | [年份范围] | 低/中/高 |
| X: [变量名] | ... | ... | ... | ... | ... |
| M: [机制变量] | ... | ... | ... | ... | ... |

**收集难度说明**：
- **低**：数据库直接下载，无需清洗
- **中**：需多表合并或简单手工补充
- **高**：需大量手工收集（如文本分析、年报摘录）

**⚠️ 风险提示**：如果任一核心变量的收集难度为"高"，必须在该研究问题后标注"数据获取风险较高，建议准备替代变量"。

---

### 3.5 内生性威胁 (Potential Pitfalls)
犀利地指出该策略最大的漏洞：
- **遗漏变量偏误 (Omitted Variable Bias)**：哪些不可观测因素同时影响 X 和 Y？
- **样本自选择 (Self-Selection)**：处理组选择是否内生？
- **反向因果 (Reverse Causality)**：Y 是否可能反过来影响 X？
- **测量误差 (Measurement Error)**：关键变量是否存在系统性测量偏误？
- **外部有效性 (External Validity)**：结论能否推广到其他情境？

**提出缓解建议**：如寻找工具变量、使用固定效应、进行安慰剂检验等。

## 注意事项

1. **学科普适性**：本技能适用于经济学、管理学、政治学、社会学、公共政策、教育学等所有社会科学领域。不同学科的识别策略偏好可能不同，请根据材料背景调整。

2. **严谨但不过度技术化**：解释计量方法时，用直觉语言说明「为什么这个方法能识别因果」，避免堆砌公式。

3. **基于材料，不臆造**：所有研究问题必须源于材料中的事实或讨论，不要凭空捏造材料中未提及的变量或关系。

4. **鼓励批判性思维**：在「审稿人视角」部分，要像 R&R 阶段的审稿人一样犀利，帮助用户提前预见可能的批评。

## 示例输出格式

```markdown
### 📌 材料核心洞察

材料聚焦于 2018 年环保税改革对重污染企业创新行为的影响。核心现象是：环保税实施后，部分企业增加了绿色专利申请，但也有企业选择搬迁而非创新。微观主体为 A 股上市公司，核心结果为绿色创新产出（专利数）和企业生产率（TFP）。

### 💡 核心研究问题与假设

**RQ1: 环保税改革是否促进了企业绿色创新？**

- **研究假设**: H1: 环保税实施显著提高了重污染企业的绿色专利申请数量
- **识别策略**: 多期双重差分 (Staggered DiD)
  - Treatment: 重污染行业企业（根据《上市公司环保核查行业分类管理名录》定义）
  - Control: 非重污染行业企业
  - 识别假设：平行趋势——若无政策，处理组与对照组创新趋势应相同
  - 模型：GreenPatent_it = α + β·Post_t × Treat_i + γX_it + μ_i + λ_t + ε_it
- **所需数据**: 
  - Y: 企业绿色专利申请数（CNIPA 专利数据库，需手工识别绿色 IPC 分类）
  - X: 环保税虚拟变量（2018 年及以后=1）× 重污染行业虚拟变量
  - 控制变量：企业规模、年龄、ROA、杠杆率、现金流（CSMAR）
- **审稿人视角 (威胁与防御)**: 
  - 威胁：2018 年可能同时出台其他影响创新的政策（如研发加计扣除）
  - 防御：加入行业×时间趋势控制；进行事件研究法检验平行趋势；安慰剂检验（随机分配处理时间）

**RQ2: 环保税影响的异质性——企业所有制的调节作用**

- **研究假设**: H2: 环保税对绿色创新的促进作用在国有企业中更弱
- **识别策略**: 三重差分 (DDD)
  - 在 DiD 基础上加入 SOE 虚拟变量及其交互项
