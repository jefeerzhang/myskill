# 开题报告评审专家 (Proposal Review Expert)

## 📋 技能概述

**分级学术评审专家** —— 根据学位层级（本科/硕士/博士）动态调整评审标准，通过精准的"降维/升维"打击与指导，帮助学生完善开题报告。

**核心定位**：你不是简单挑刺，而是根据学生所在学历层次，提供匹配的评审标准和建设性修改建议。

---

## ✨ 核心功能

### 1. 学位层级动态调整
| 层级 | 评审标准 | 创新性要求 | 方法论要求 | 内生性要求 |
|------|---------|-----------|-----------|-----------|
| **本科** | 规范性 + 逻辑闭环 | 允许应用成熟理论 | 基础模型（OLS/案例框架） | 高容忍，了解即可 |
| **硕士** | 文献对话 + 方法严密 | 理论视角补充/机制深挖 | 固定效应 + 基础内生性处理 | 必须讨论，可用滞后项/控制变量 |
| **博士** | 理论贡献 + 因果识别 | 显著边际贡献 | 严谨因果识别（IV/DID/RD/PSM） | 必须严谨处理，否则一票否决 |

### 2. PDF 自动提取
- 支持直接上传开题报告 PDF 文件
- 使用 MinerU 自动提取文本内容
- 提取失败时自动降级使用其他方法

### 3. 核心变量界定审查（强制）
**铁律**：必须审查解释变量 (X)、被解释变量 (Y)、中介变量 (M) 的定义是否准确、逻辑是否自洽。若发现概念性错误，必须列为"致命伤"。

**审查维度**：
1. 概念 - 操作一致性
2. 变量构造逻辑（是否机械加总）
3. 内生性来源（变量构造是否引入系统性偏误）
4. 文献对照（是否与主流文献一致）

### 4. RAG 文献专项验证（推荐）
- 基于 AnythingLLM 知识库进行智能文献检索
- 5 个精准查询替代泛化查询
- **重要**：RAG 结果仅供参考，不是评判的绝对标准

### 5. 五维动态评估
1. 选题价值 (Value)
2. 文献综述 (Literature)
3. 研究设计 (Methodology)
4. 逻辑框架 (Logic)
5. 可行性 (Feasibility)

### 6. 致命伤分级预警
- **本科致命伤**：逻辑断裂、题目过大、格式不规范、核心变量界定错误
- **硕士致命伤**：伪创新、变量测量误差、变量构造概念混淆、无法识别因果
- **博士致命伤**：无理论贡献、内生性未处理、机制检验缺失、变量操作化与理论脱节

### 7. 评审报告自动保存
- 自动保存到 `./开题报告评审/` 目录
- 文件名格式：`<学位层级>_<日期>_<关键词>.md`

```
[生成评审报告并自动保存]
```

## ⚙️ RAG 配置（可选）

### 为什么要配置 RAG？
RAG 检索可以验证学生引用文献的完整性，识别缺失的重要文献，提升评审报告质量。但**RAG 结果仅供参考，不是评判的绝对标准**。

### 配置方法

#### 方法 1：在评审时提供
```
用户：评审这份开题，启用 RAG
- AnythingLLM Base URL: http://localhost:3001
- API Key: your-api-key
- Workspace Slug: your-workspace-id

[粘贴开题报告内容]
```

#### 方法 2：预先配置（推荐）
在技能同目录下创建 `.rag_config.json` 文件：
```json
{
  "base_url": "http://localhost:3001",
  "api_key": "your-api-key",
  "workspace_slug": "your-workspace-id"
}
```

评审时只需说"启用 RAG"即可自动读取配置。

### 获取配置信息
1. 打开 AnythingLLM 桌面应用
2. 进入设置 → API 设置，生成 API Key
3. 工作区 Slug 可在 URL 或工作区设置中获取

## ⚠️ 重要说明

### 关于 RAG 检索
- **RAG 检索结果仅供参考**，不是评判开题报告质量的绝对标准
- **知识库覆盖度有限**：RAG 结果受限于知识库中的文献，可能遗漏领域重要文献
- **不可替代独立判断**：评审应基于学术规范和领域常识进行独立评判
- **RAG 未命中≠学生错误**：可能是知识库缺失，而非学生引用不当
- **RAG 命中≠学生必须引用**：学生选题可能有创新性，不必强制引用

### 关于核心变量审查
- 若发现变量界定存在**概念性错误**（如将政府补助纳入社会资本），必须列为"致命伤"
- 这是基于学术理论定义的基本要求，与学位层级无关

### 关于评审结论
- 结论必须考虑学位层级（如"作为硕士开题勉强合格，但作为博士开题需重新设计"）
- 不可用同一标准评审所有开题

## 🎯 适用场景

**适用**：
- 本科/硕士/博士开题报告评审
- 研究计划审核
- 论文初稿评审（可调整标准）
- 学术写作指导

**不适用**：
- 期刊论文同行评审（使用 peer-review 技能）
- 一般论文质量评估（使用学术写作技能）
- 快速摘要提取（使用 summarize 技能）

## 📚 示例案例

### 案例 1：本科开题（社会资本与家庭金融资产）
**评审发现**：
- ✅ 文献综述基础较好，引用了关键文献
- ✅ 方法设计合理（双向固定效应+CFPS 数据）
- ❌ 解释变量界定错误：将政府补助纳入社会资本（概念混淆）
- ❌ H1 推导的理论机制路径不聚焦
- ❌ 稳健性检验设计较单薄

**结论**：建议微调后开题（需修正变量定义）

### 案例 2：硕士开题（REITs 估值溢价）
**评审发现**：
- ✅ 选题具有时效性和研究价值
- ✅ 数据基础扎实（78 只全样本）
- ❌ 研究假设未独立陈述
- ❌ 内生性处置逻辑薄弱

**结论**：建议微调后开题

## 🔧 故障排查

| 问题 | 解决方案 |
|------|---------|
| PDF 提取失败 | 检查 MinerU 是否安装；尝试使用 pdf 技能替代 |
| RAG 检索返回 404 | 检查工作区 Slug 是否正确 |
| RAG 检索返回 401 | 检查 API Key 是否有效 |
| RAG 检索超时 | 检查 AnythingLLM 服务是否运行 |
| 检索结果为空 | 确认知识库中已有相关文献 |
| 检索结果不相关 | 优化查询关键词或补充领域文献 |
