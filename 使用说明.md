# 📖 文献导读助手 - 使用说明与最佳实践

**版本**：2.0.0  
**最后更新**：2026-03-27

---

## 🎯 使用场景

### 适合的情况 ✅

- 快速理解新领域的核心论文
- 准备文献综述或开题报告
- 需要把握论文的核心贡献
- 跨学科阅读（非本专业论文）
- 教学或组会分享准备

### 不适合的情况 ❌

- 需要精读每一处细节（建议手动阅读）
- 完全依赖 AI 而不自己思考
- 期望 100% 准确（AI 可能出错）

---

## 📊 工作流程

```
用户提供 PDF/链接
    ↓
检查 MinerU CLI 是否安装
    ↓
自动选择提取模式（flash vs extract）
    ↓
提取 PDF 内容为 Markdown
    ↓
执行文献 DNA 扫描
    ↓
生成四层递进导读
    ↓
保存报告到 ./文献导读/
```

## 🔧 最佳实践

### 1. 文件准备

**推荐**：
- PDF 文件 < 50MB
- 分辨率适中的扫描件（非图片格式）
- 文字版 PDF（非纯图片扫描）

**不推荐**：
- 加密的 PDF（需先解密）
- 纯图片扫描件（OCR 准确率低）
- 超大文件（>100MB）

### 2. 模式选择

**flash-extract（快速模式）**：
- 适合：文字版 PDF、简单排版
- 速度：5-10 秒/页
- 成本：免费

**extract（精确模式）**：
- 适合：复杂公式、表格、图表
- 速度：10-30 秒/页
- 成本：需要 API Token

### 3. 读者层级选择

技能会询问你的研究背景，如实回答以获得最佳体验：

- **科研小白**：大量生活类比，术语极少
- **进阶人员**：适度术语，平衡深浅
- **资深学者**：直接进入技术细节

### 4. 报告使用

**正确姿势**：
1. 先快速浏览四层导读，把握整体
2. 对照原文验证关键论断
3. 使用自测清单检查理解程度
4. 参考知识族谱图延伸阅读

**错误姿势**：
- 完全依赖导读，不读原文
- 直接引用导读内容而不核验
- 忽视批判层的局限性分析

## ⚠️ 注意事项

### 1. 准确性

- 技能会进行 Web Search 验证关键论断
- 但仍建议核验重要结论
- 不确定内容会标注"[待核验]"

### 2. 隐私

- PDF 文件会上传到 MinerU API 处理
- 敏感文件（未发表/机密）慎用
- 可选择本地提取工具替代

### 3. 配额限制

- flash-extract：免费，无限制
- extract：依赖 API Token 配额
- 如配额用尽，等待下月重置

## 🛠️ 故障排查

### 问题 1: 技能无响应

**检查**：
```bash
# 验证 MinerU 安装
mineru-open-api version

# 验证技能目录
ls ~/Library/Application\ Support/LobsterAI/SKILLs/academic-literature-guide-v2/
```

**解决**：
1. 重启 LobsterAI
2. 重新安装技能
3. 检查 SKILL.md 语法

### 问题 2: 提取内容为空

**可能原因**：
- PDF 加密或损坏
- 纯图片扫描件
- 网络连接问题

**解决**：
1. 检查 PDF 是否可正常打开
2. 尝试 extract 模式（支持 OCR）
3. 检查网络连接

### 问题 3: 公式显示为乱码

**原因**：Markdown 阅读器不支持 LaTeX

**解决**：
- 使用支持 LaTeX 的编辑器（VS Code / Typora / Obsidian）
- 安装 Markdown All in One 插件（VS Code）

## 🎓 进阶技巧

### 1. 批量处理

可以一次性提供多个 PDF 文件，技能会依次处理：
```
帮我解读这 3 篇 PDF：
- paper1.pdf
- paper2.pdf
- paper3.pdf
```

### 2. 指定读者层级

可以直接指定：
```
用资深学者的视角解读这篇论文：paper.pdf
```

### 3. 关注特定方面

可以要求侧重某个层面：
```
重点讲解技术层的方法细节：paper.pdf
```

### 4. 追溯后续研究

可以询问：
```
这篇论文发表后有哪些后续研究？
有没有复现或批评的文章？
```
