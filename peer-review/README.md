# 同行评议（peer-review）v2.0

面向学术论文与基金申请的**同行评审/专家评审**技能：用结构化检查清单做系统评估，输出可直接用于“审稿报告 / 修改建议 / 回复审稿意见（Response/Rebuttal）”的成品文本。

## 🎯 这个技能解决什么问题？

当你需要完成下列任务时，用 `peer-review`：

- 写一份像期刊审稿人那样的正式评审意见（Major/Minor、可接收/需大修/拒稿）
- 对论文或基金申请书做方法论与统计审查（尤其人文社科计量/因果识别）
- 基于审稿意见生成“逐条回复信”与“修改说明”，并给出可执行的返工清单
- 对照报告规范清单（如 CONSORT/STROBE/PRISMA 等）检查合规性

## 🔀 与 proposal-review-expert 的区别（最重要）

| 维度 | proposal-review-expert（开题评审） | peer-review（同行评议） |
| --- | --- | --- |
| 评审对象 | 开题报告 / 研究计划 / Proposal | 已完成论文稿件 / 投稿稿 / 修改稿 / 基金申请书 |
| 研究阶段 | 研究开始前（计划阶段） | 研究完成后（投稿/修改阶段） |
| 核心问题 | “这项研究值得做吗？设计可执行吗？” | “证据够不够？方法与统计站得住吗？怎么改到可发表/可通过？” |
| 输出风格 | 导师式分层评审：致命伤预警 + 重构建议 | 审稿人式可交付：审稿报告 + 逐条回复 + 合规清单 |
| 触发关键词（建议） | 开题/答辩/研究计划/Proposal/课题申请/硕士博士层级 | 同行评议/同行评审/审稿意见/Response/Rebuttal/投稿/修改说明 |

## ✅ 触发方式

在对话中明确以下任一意图即可触发：

- “请对这篇论文进行同行评议/同行评审”
- “我收到了审稿意见，帮我写 Response to Reviewers / rebuttal”
- “请以期刊审稿人/基金评审专家视角评估这份稿件/申请书”

## 📥 输入建议（给出这些信息，效果最好）

- 稿件全文或关键章节（摘要/引言/方法/结果/讨论）
- 研究类型：实证/理论/综述/实验/调查
- 目标期刊/会议（如有）或目标基金（国自然/社科基金等）
- 已收到的审稿意见（若处于返修阶段）
- 你希望的输出：审稿报告 / 返工清单 / 逐条回复信 / 规范合规核查

## 📤 输出会长什么样（建议结构）

- Overall Summary（1–2 段）：论文贡献与主要问题
- Recommendation：Accept / Minor / Major / Reject（或基金：建议资助/不建议）
- Major Comments：优先级最高、必须改的点（方法、识别、数据、结论力度、可重复性）
- Minor Comments：措辞、格式、补充说明、图表与引用
- Checklist（可选）：因果识别/内生性/稳健性/机制/异质性/数据审查 + 报告规范清单
- Response to Reviewers（如需要）：逐条引用意见 → 回复 → 修改位置 → 具体改法

## 🧩 典型工作流（怎么用得最顺）

### 工作流 A：写一份“可投递”的同行评审报告

1) 你提供：论文全文（或至少 摘要+引言+方法+主要结果）  
2) 你说明：目标期刊/领域、期望评审口径（偏严格/偏建设性）  
3) 产出：Summary + Recommendation + Major/Minor + 返工清单

示例指令：

```
请对这篇论文进行同行评议。输出：Summary、Recommendation、Major Comments、Minor Comments、返工清单。
[粘贴论文内容或上传 PDF]
```

### 工作流 B：写“逐条回复审稿意见（Response/Rebuttal）”

1) 你提供：审稿意见全文（最好带 Reviewer #1/#2）+ 你当前稿件版本的关键段落  
2) 产出：中英文回复信 + 每条意见的“改法建议” + 修改位置占位（你填页码行号）

示例指令：

```
我收到了审稿意见，帮我写逐条回复（中英文各一份），并给出每条意见应该怎么改的具体方案。
[粘贴审稿意见]
```

### 工作流 C：基金申请书评审（国自然/社科基金）

1) 你提供：科学问题、研究内容、技术路线、创新点、研究基础  
2) 产出：按评审要点的“扣分项清单” + “可执行的重写建议”

示例指令：

```
请以基金评审专家视角评估这份申请书，重点检查：科学问题属性、技术路线可行性、创新点与研究基础。
[粘贴申请书关键内容]
```
